## 📱 Two Ways to Get MÜN OS on Your Phone

### Option A: PWA (Progressive Web App) — INSTANT

Your app is already a PWA! This is the fastest way:

#### Android:
1. Open Chrome on your phone
2. Navigate to your MÜN OS URL
3. Tap the menu (⋮) → "Add to Home screen"
4. Tap "Install"
5. ✅ MÜN OS is now on your home screen like a native app!

#### iOS (iPhone):
1. Open Safari on your iPhone
2. Navigate to your MÜN OS URL
3. Tap the Share button (square with arrow)
4. Scroll down and tap "Add to Home Screen"
5. Name it "MÜN OS" and tap "Add"
6. ✅ MÜN OS is now on your home screen!

**PWA Features:**
- ✅ Full screen experience (no browser UI)
- ✅ Offline capability (basic features work without internet)
- ✅ Push notifications ready (when configured)
- ✅ Auto-updates when you deploy new versions
- ✅ Icon on home screen

### Option B: APK (Android App) — Build Required

For a true Android app, you need to build the APK:

#### Prerequisites:

1. **Install Android Studio** from https://developer.android.com/studio
2. **Set environment variables:**
   ```bash
   export ANDROID_HOME=$HOME/Android/Sdk
   export PATH=$PATH:$ANDROID_HOME/emulator
   export PATH=$PATH:$ANDROID_HOME/platform-tools
   ```
3. **Install JDK 17:**
   ```bash
   # macOS
   brew install openjdk@17
   
   # Linux
   sudo apt install openjdk-17-jdk
   
   # Windows
   # Download from adoptium.net
   ```

#### Build Steps:

```bash
# Navigate to project
cd /home/z/my-project

# Quick build (opens Android Studio)
./scripts/build-apk.sh --prepare

# Or build debug APK directly
./scripts/build-apk.sh --debug

# APK will be at:
# download/mun-os-debug.apk
```

#### Install on Phone:

1. Transfer `mun-os-debug.apk` to your phone
2. Open the file on your phone
3. Allow installation from unknown sources if prompted
4. Install and enjoy!

## 📋 Signing for Release

To publish on Play Store or install without "unknown sources" warning:

1. Generate a signing key:
   ```bash
   keytool -genkey -v -keystore mun-os.keystore \
     -alias mun-os -keyalg RSA -keysize 2048 -validity 10000
   ```

2. Create `android/key.properties`:
   ```properties
   storePassword=YOUR_PASSWORD
   keyPassword=YOUR_PASSWORD
   keyAlias=mun-os
   storeFile=../mun-os.keystore
   ```

3. Update `android/app/build.gradle`:
   ```gradle
   android {
       signingConfigs {
           release {
               def props = new Properties()
               props.load(new FileInputStream(rootProject.file("key.properties")))
               storeFile file(props.storeFile)
               storePassword props.storePassword
               keyAlias props.keyAlias
               keyPassword props.keyPassword
           }
       }
       buildTypes {
           release {
               signingConfig signingConfigs.release
           }
       }
   }
   ```

## 💡 Tips

1. **PWA is usually enough** - Most users don't need a native APK
2. **Debug APK is for testing** - Shows logs, not optimized
3. **Release APK is for distribution** - Optimized, smaller, faster
4. **Keep both PWA and APK** - Some users prefer each method

## 🆘 Troubleshooting

### "ANDROID_HOME not set"
Set the environment variable to your Android SDK path.

### "Gradle build failed"
Open Android Studio and build from there for better error messages.

### "White screen on app launch"
Check that `next.config.mobile.ts` is using `output: "export"` and `images.unoptimized: true`.

### PWA not installing
Make sure you're using HTTPS (required for PWA). The tunnel URL should work.
